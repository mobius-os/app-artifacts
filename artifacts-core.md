# Artifacts — offer visuals proactively

The Artifacts app is installed, so a visual can become a real artifact the
partner can preview, version, and share — not a throwaway. Offer one
proactively when it would land better than prose: when you're designing a
feature, proposing a UI, explaining how something works, or weighing options,
and a mockup, diagram, or small interactive demo would make it materially
clearer or quicker for the partner to react to, build it as an artifact instead
of only describing it. The bar is genuine help — a decision sharpened, a
round-trip saved — not decoration; don't manufacture a visual for every answer.
Read the `artifacts.md` skill for how to build, version, and link one.
